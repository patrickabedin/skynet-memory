# Schema Reference

## Table: `agent_memories`

| Column | Type | Description |
|---|---|---|
| `id` | TEXT PK | SHA-256(source + content).slice(0,12) — dedup key |
| `content` | TEXT | Raw memory text |
| `source` | TEXT | Origin path/identifier (e.g. `memory/2026-03-12.md`) |
| `source_type` | TEXT | See source types below |
| `entities` | TEXT[] | Named entities: coins, strategies, functions, people |
| `topics` | TEXT[] | Broad categories: trading, bug, strategy, risk, market, config |
| `tags` | TEXT[] | Freeform: recurring, async, critical, SL, ratchet, etc. |
| `importance` | FLOAT | 0.0–1.0 — how significant for future decisions |
| `consolidated` | BOOLEAN | false = needs ConsolidateAgent processing |
| `promoted` | BOOLEAN | true = written to MEMORY.md |
| `connections` | JSONB | `[{ to_id, relationship }]` — links to related entries |
| `embedding` | vector(1536) | text-embedding-3-small vector |
| `created_at` | TIMESTAMPTZ | Ingest timestamp |
| `consolidated_at` | TIMESTAMPTZ | When ConsolidateAgent processed this entry |
| `promoted_at` | TIMESTAMPTZ | When promoted to MEMORY.md |

## Source Types

| source_type | Description | Example source |
|---|---|---|
| `daily_log` | From memory/YYYY-MM-DD.md | `memory/2026-03-12.md` |
| `trade_outcome` | Closed trade from Supabase | `supabase:v4_trades:row_123` |
| `error_log` | From .learnings/ERRORS.md | `.learnings/ERRORS.md:ERR-042` |
| `session_export` | Auto-exported from conversation | `session:main:2026-03-12T23:00` |
| `manual` | Manually ingested via API | `manual` |
| `insight` | Generated by ConsolidateAgent | `instincts.json:insight_017` |
| `archive_summary` | Compressed old entries | `archive:2026-01` |

## Importance Scoring Guide

| Score | Meaning | Examples |
|---|---|---|
| 0.9+ | Critical — must remember | Large loss, recurring bug class, major architecture decision |
| 0.7–0.9 | High — auto-promote to MEMORY.md | SL hit, new instinct confirmed, strategy change |
| 0.5–0.7 | Medium — consolidate and connect | Normal trade outcome, config change, minor fix |
| 0.3–0.5 | Low — keep but don't promote | Routine log entries, minor observations |
| < 0.3 | Noise — candidate for decay | Short entries, duplicates, low-signal observations |

## SQL Functions

### `match_memories(query_embedding, match_threshold, match_count, filter_topics, filter_tags)`
Semantic similarity search using cosine distance. Returns entries with similarity > threshold.

### `search_memories_by_topic(search_topics, min_importance, result_limit)`
Topic-based search without embedding. Fast, no LLM call needed.
